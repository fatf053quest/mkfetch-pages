# mkfetch-pages

一个用于归档和发布独立 HTML 页面的仓库。  
本仓库不面向特定网站或域名，旨在以通用方式整理和存放多个独立的网页内容。

## 项目简介

`mkfetch-pages` 是一个轻量级的页面归档仓库，用于存放由不同来源或手动生成的 HTML 文件。  
你可以将其视为一个静态页面集合，每个文件均为独立的网页，可直接通过浏览器打开或部署至支持静态页面的服务。

## 目录说明

```
.
├── index.html          # 仓库入口页面（可选）
├── pages/              # 存放归档 HTML 页面的目录
│   ├── example1.html
│   ├── example2.html
│   └── ...
├── assets/             # 页面所需的资源文件（CSS、JS、图片等）
│   ├── style.css
│   └── ...
└── README.md           # 本文件
```

- **pages/**：主要归档目录，所有独立 HTML 页面放置于此。
- **assets/**：存放页面引用的外部资源，按需组织。
- **index.html**：可选，可作为仓库的导航页或入口。

## 页面归档说明

- 每个 HTML 文件应保持独立，尽量不依赖外部资源，或仅依赖 `assets/` 目录下的文件。
- 页面命名建议使用英文或拼音，避免特殊字符。
- 如需引用其他页面，请使用相对路径（如 `./pages/example.html`）。
- 本仓库不限制页面内容类型，但请遵守相关法律法规及平台使用政策。

## 维护说明

- 欢迎提交 PR 添加新的独立 HTML 页面，或修正现有页面问题。
- 添加页面时，请确保其功能正常、资源路径正确。
- 不建议在页面中包含跟踪脚本、广告或其他可能影响用户隐私的内容。
- 本仓库的维护者会定期检查归档内容，清理无效或损坏的页面。

## 使用建议

- 可直接通过 GitHub Pages 或其他静态托管服务部署本仓库。
- 若需批量处理页面，可参考仓库中的脚本文件（如有）。
- 对于大型页面或复杂资源，建议压缩后提交。
